Class InputMangler

Maps keyboard/gamepad inputs and modifier keys/buttons to commands.

"modifiers" are keys and/or buttons held down at the same time as the main input is pressed. This allows input chords like CTRL+q to be mapped to a different command than just q.

Mapping may optionally be restricted to a named context (in other words, you can have a "game" context, a "paused" context, a "menu" context, etc., and the same key or button will return a different command depending on context).

InputMangler:addContext (context)
Creates a new context. Warning: If there is already a context named "context", calling this will blank it.

Parameters:

  • context name of the new context (string)
InputMangler:delContext (context)
Deletes the named context.

Parameters:

  • context name of the context to be deleted (string)
InputMangler:addModifierKey (modifier, context)
Adds a modifier key to the specified context. Note: Adding the same modifier twice won't actually break anything (just slow things down slightly when the keys are tested).

Parameters:

  • modifier a KeyConstant
  • context name of the context to apply this modifier to (or nil to apply it to the default context)
InputMangler:addModifierButton (modifier, context)
Adds a modifier button to the specified context. Note: Adding the same modifier twice won't actually break anything (just slow things down slightly when the keys are tested).

Parameters:

  • modifier a GamepadButton
  • context name of the context to apply this modifier to (or nil to apply it to the default context)
InputMangler:getModifiers (context)
Returns the modifier keys and buttons that are currently being held down.

Parameters:

  • context The name of an existing context (string) containing the modifier list to test.

Returns:

    list of held modifiers (table containing a mix of KeyConstants and GampabButtons) or nil if no modifiers are being held.
InputMangler:isKeyModifier (keysym, context)
Tests whether or not the given key is a modifier. return True

Parameters:

  • keysym The KeyConstant to test
  • context The name of an existing context (string) to test this key in (or nil to use the default context)
InputMangler:isButtonModifier (button, context)
Tests whether or not the given button is a modifier. return True

Parameters:

  • button The GamepadButton, GamepadAxis, or numeric mouse button identifier to test
  • context The name of an existing context (string) to test this key in (or nil to use the default context)
InputMangler:addKeyCommand (command, keysym, context, modifiers)
Adds a key binding to the specified context.

Parameters:

  • command command name (string)
  • keysym KeyConstant to bind the command to
  • context the name of an existing context (string) to place this command in (or nil to use the default context)
  • modifiers a list of modifier keys or buttons to complete the input chord (or nil for no modifiers)
InputMangler:delKeyCommand (keysym, context, modifiers)
Removes a key binding from the specified context.

Parameters:

  • keysym the KeyConstant to remove
  • context the name of an existing context (string) to place this command in (or nil to use the default context)
  • modifiers a list of modifier keys or buttons to complete the input chord (or nil for no modifiers)
InputMangler:addButtonCommand (command, button, context, modifiers)
Adds a button, axis, or arbitrary binding to the specified context.

Parameters:

  • command command name (string)
  • button a GamepadButton or arbitrary input name (string) to bind the command to
  • context the name of an existing context (string) to place this command in (or nil to use the default context)
  • modifiers a list of modifier keys or buttons to complete the input chord (or nil for no modifiers)
InputMangler:delButtonCommand (button, context, modifiers)
Removes a button, axis, or arbitrary binding from the specified context.

Parameters:

  • button a GamepadButton or arbitrary input name (string) to remove
  • context the name of an existing context (string) to place this command in (or nil to use the default context)
  • modifiers a list of modifier keys or buttons to complete the input chord (or nil for no modifiers)
InputMangler:testButton (button, context)
Returns the command bound to the specified GamepadButton or arbitrary input, taking modifiers into account.

Parameters:

  • button a GamepadButton or arbitrary input name (string) to test
  • context the name of an existing context (string) to check (or nil to use the default context)

Returns:

    a command (string) if a mapping for the button exists in the specified context, nil if it doesn't.
InputMangler:testKey (keysym, context)
Returns the command bound to the specified KeyConstant, taking modifiers into account.

Parameters:

  • keysym a KeyConstant to test
  • context the name of an existing context (string) to check (or nil to use the default context)

Returns:

    a command (string) if a mapping for the key exists in the specified context, nil if it doesn't.
InputMangler:saveBindings (filename, rawio)
Saves all bindings and modifiers to a file.

Parameters:

  • filename the name of a file
  • rawio set to true in order to bypass love.filesystem and use standard Lua io routines.
InputMangler:loadBindings (filename, rawio)
Loads bindings and modifiers from a file saved with the saveBindings function. Warning: this will wipe away any modifiers or bindings already in teh system.

Parameters:

  • filename the name of a file
  • rawio set to true in order to bypass love.filesystem and use standard Lua io routines.
InputMangler:setJoystick (joystick)
Sets the gamepad to test for modifier buttons. You only need to call this if you want to test a gamepad other than the one that was love.joysticks.getJoysticks[1] when the module was initialized.

Parameters:

  • joystick a love.joysticks.Joystick to test for modifier buttons, or nil to not test any
